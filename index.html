# Workshop Inventory — Mobile Web App (client‑only, local list) This is a **stand‑alone web app** (a single `index.html`) designed for **phones** on the shop floor. It does exactly one thing: 1. **Scans** barcodes (**EAN‑13/EAN‑8/Code‑128**). 2. **Adds** each scan to a **local list** (persisted in `localStorage`). 3. Lets you **edit quantities** (+/‑, delete). 4. **Submit**: produces a **JSON payload** (copy/share/download) to send wherever you want later (Excel/Server integration can come later). > No backend required. On GitHub Pages you’ll automatically have **HTTPS**, which is required to use the **camera** on mobile. --- ## Single file — `index.html` > **Note:** fully working code below (HTML+CSS+JS). It includes ZXing via CDN, minimal styling, scan loop, quantity edits, and export (copy/share/download). Save as `index.html` and push to a GitHub repo with Pages enabled. ```html Inventory — Scanner

Inventory — Scanner

(Optional) will be included in the payload.
Items: 0
Total qty: 0

Payload ready

Copy, share, or download the JSON. Then you can clear the list.
``` --- ## Deploy on GitHub Pages (quick) 1. Create a public repo (e.g., `workshop-inventory-scanner`) and add this **`index.html`**. 2. Commit & push to `main`. 3. Repo → **Settings → Pages** → *Deploy from a branch* → Branch `main` → Folder `/ (root)` → **Save**. Your URL will be `https://.github.io/workshop-inventory-scanner/`. That’s it — open the URL on your phone, tap **Start scanner**, and you’re set. --- ## Next (optional) * **PWA**: installable app icon + offline cache. * **Local CSV mapping** (barcode→name/unit) shown client‑side. * **Automatic submit hook** to an API when you’re ready to integrate with Excel.
